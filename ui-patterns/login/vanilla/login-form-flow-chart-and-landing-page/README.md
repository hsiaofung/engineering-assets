## 登入表單流程圖與著陸頁面重點整理

### 1. 建立表單前，先規劃完整流程

不要急著開始寫 HTML 或 CSS，而是先思考整個使用者流程（User Flow）。

應先確認：

* 使用者如何開始操作？
* 成功登入後要去哪裡？
* 登入失敗怎麼處理？
* 忘記密碼怎麼辦？
* 新使用者如何完成註冊？

> **重點：先設計流程，再設計畫面。**

---

### 2. 使用流程圖（Flow Chart）釐清所有情境

流程圖能幫助設計者確認是否涵蓋所有可能的操作路徑。

基本登入流程如下：

```
進入網站
    │
    ├── 已有帳號
    │      │
    │      ├── 輸入帳號密碼
    │      │      │
    │      │      ├── 成功 → 登入系統
    │      │      └── 失敗
    │      │             │
    │      │             ├── 忘記密碼 → 重設密碼
    │      │             └── 重新登入
    │
    └── 沒有帳號
           │
           ├── 註冊
           ├── 驗證資料
           └── 回到登入流程
```

流程圖可以避免遺漏：

* Error Flow（錯誤流程）
* Success Flow（成功流程）
* Edge Cases（特殊情境）

---

### 3. 設計登入前的著陸頁（Landing Page）

使用者通常不是一進網站就看到登入表單，而是先看到一個著陸頁。

著陸頁的目的：

* 介紹產品或服務
* 說明加入的價值
* 引導使用者採取下一步行動

這就是所謂的 **CTA（Call to Action）**。

例如：

```
Become the Change

[ Be the Change ]
[ Login ]
```

其中：

* 已有會員 → Login
* 新使用者 → Be the Change（註冊）

---

### 4. 讓使用者清楚知道下一步

每個畫面都應明確告訴使用者現在要做什麼。

例如：

* Login
* Register
* Forgot Password
* Reset Password

避免讓使用者猜測流程。

---

### 5. 登入與註冊可先分流

不要一開始就在同一頁塞滿所有表單。

建議流程：

```
Landing Page
      │
      ▼
Choose Login / Register
      │
      ├── Login Page
      └── Register Page
```

這樣能降低認知負擔，讓畫面更乾淨、更容易理解。

---

### 6. 先完成流程設計，再開始製作表單

在流程確認後，再開始：

* 設計 UI
* 撰寫 HTML
* 加入 CSS
* 串接 API
* 處理驗證

這樣能避免後續因流程變更而大量修改程式碼。

---

## 前端開發實務建議

在開始實作登入功能前，可以先準備：

* ✅ User Flow（登入流程圖）
* ✅ Wireframe（頁面草圖）
* ✅ Landing Page（著陸頁）
* ✅ Login Page（登入頁）
* ✅ Register Page（註冊頁）
* ✅ Forgot Password Page（忘記密碼）
* ✅ Reset Password Page（重設密碼）
* ✅ Success／Error 狀態設計
* ✅ API Flow（登入、註冊、重設密碼流程）
* ✅ Route Flow（頁面導航）

---

## 核心觀念

* **流程優先於畫面**：先設計完整的使用者流程，再開始實作 UI。
* **流程圖能降低遺漏**：涵蓋成功、失敗與例外情境，避免開發後期才補流程。
* **著陸頁提升轉換率**：透過清楚的 CTA，引導使用者登入或註冊。
* **一步一步引導使用者**：每個頁面只聚焦一個主要任務，降低操作負擔。
* **規劃完成再開發**：先確認流程與頁面架構，再進行 UI 與 API 實作，可減少返工並提升開發效率。
