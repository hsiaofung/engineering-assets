# 建立與樣式化註冊頁面重點整理

## 1. 延續既有設計，保持一致性

註冊頁面與登入頁採用相同的設計語言，包括：

* 相同的表單元件
* 相同的配色
* 相同的排版風格
* 相同的密碼可見性切換功能

不同之處在於：

* 欄位數量較多
* 採用兩欄式版面（桌面版）
* 增加歡迎資訊區塊

> **設計原則：Consistency（一致性）**
> 使用者不需要重新學習介面，提高操作效率。

---

## 2. 重複利用既有樣式

課程沒有重新建立一套 CSS，而是在既有的 `form.css` 基礎上擴充。

做法包括：

* 重用表單元件樣式
* 重用按鈕樣式
* 重用密碼顯示／隱藏功能
* 僅新增註冊頁特有的樣式

> **最佳實務：遵循 DRY（Don't Repeat Yourself）原則，避免重複撰寫相同樣式。**

---

## 3. 採用 Mobile First 設計

課程先設計手機版，再逐步擴充桌面版。

### 手機版

```text
+----------------------+
|      Welcome         |
+----------------------+
|                      |
|   Sign Up Form       |
|                      |
+----------------------+
```

特色：

* 單欄排列
* 方便閱讀
* 適合小螢幕

---

### 桌面版

```text
+--------------+----------------+
|              |                |
|   Welcome    |  Sign Up Form  |
|              |                |
+--------------+----------------+
```

特色：

* 左右兩欄
* 更有效利用寬螢幕空間
* 提供更好的資訊層次

---

## 4. 使用 CSS Grid 建立兩欄版面

桌面版透過 **CSS Grid** 將版面切換為兩欄：

* 左側：Welcome 區塊
* 右側：註冊表單

優點：

* 排版清楚
* 容易控制欄寬
* 響應式調整更容易

---

## 5. Border Radius 的設計

課程依不同裝置調整圓角。

### 手機版

先移除圓角：

* 適合上下堆疊版面
* 視覺較自然

### 桌面版

再依區塊加入不同圓角：

```text
+-----------+-----------+
|           |           |
| Welcome   | Form      |
|           |           |
╰───────────┴───────────╯
```

例如：

* Welcome：左下圓角
* Form：右下圓角

讓兩個區塊看起來像一個完整卡片。

---

## 6. 利用 Specificity 覆蓋共用樣式

大部分按鈕樣式沿用登入頁，但註冊按鈕需要不同配色，因此利用 **CSS Specificity（特異性）** 覆蓋。

優點：

* 保持共用樣式
* 只修改需要差異化的部分
* 降低維護成本

---

## 7. Welcome 區塊設計

註冊頁新增 Welcome 區塊：

包含：

* 標題
* 說明文字
* 背景色

用途：

* 提供品牌形象
* 引導使用者完成註冊
* 平衡頁面視覺

---

## 8. Optional 欄位設計

課程將「選填」文字包在 `<span>` 中，並調整樣式。

例如：

```text
Phone Number (Optional)
```

樣式通常會：

* 字體較小
* 顏色較淡
* 不影響主要資訊

目的：

* 清楚標示非必填欄位
* 降低使用者填寫壓力

---

## 9. 密碼欄位間距調整

因密碼欄位後方還有其他輸入欄位，因此額外增加底部間距（Margin）。

好處：

* 避免畫面擁擠
* 提高可讀性
* 維持一致的垂直節奏（Vertical Rhythm）

---

## 10. 響應式設計流程

整體版面切換如下：

```text
手機版

Welcome
↓↓↓

Sign Up Form
```

↓

```text
桌面版

Welcome   |   Sign Up Form
```

只需透過 Media Query，即可完成版面轉換。

---

## 11. 設計理念

本課強調的不是新增大量 CSS，而是**建立可重用、可擴充的樣式架構**。

核心理念包括：

* **Consistency（一致性）**：延續登入頁的設計風格。
* **Reusability（可重用性）**：重複利用既有元件與樣式。
* **Mobile First（行動裝置優先）**：先完成手機版，再優化桌面版。
* **Specificity（特異性）**：局部覆蓋共用樣式，避免重複定義。
* **Responsive Design（響應式設計）**：依螢幕大小切換單欄與雙欄版面。

---

# 本課核心重點（適合筆記）

* 註冊頁應延續登入頁的設計語言，保持一致的使用體驗。
* 優先重用既有 `form.css` 中的共用樣式，只新增註冊頁專屬設定。
* 採用 **Mobile First** 設計，先完成單欄版面，再透過 Media Query 優化桌面版。
* 使用 **CSS Grid** 建立桌面版雙欄配置（Welcome 區塊＋註冊表單）。
* 依不同裝置調整 `border-radius`，讓整體卡片在不同版型下保持自然的視覺效果。
* 利用 **CSS Specificity** 覆蓋共用樣式，例如註冊按鈕的配色，而非重新定義所有樣式。
* Welcome 區塊可作為品牌介紹與註冊引導，提升頁面資訊層次。
* 「Optional（選填）」應透過較淡、較小的文字標示，清楚傳達非必填資訊。
* 適當調整欄位間距，維持一致的垂直節奏，提升閱讀與填寫體驗。
* **重用樣式、保持一致性、行動裝置優先**，是建立可維護且易擴充表單介面的重要原則。
