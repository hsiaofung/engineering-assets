# Form Layout Checklist

Use this checklist before submitting a design, implementing a form, or creating a pull request.

## Structure

- [ ] Don't copy paper forms directly.
- [ ] Place the form inside a dedicated container.
- [ ] Prefer a single-column layout.
- [ ] Group related fields logically.
- [ ] Arrange fields in a natural, top-to-bottom flow.

## Efficiency

- [ ] Remove unnecessary fields.
- [ ] Minimize mouse interactions.
- [ ] Match input width to expected data.
- [ ] Keep label placement consistent.
- [ ] Do not use placeholders as labels.

## Error Handling

- [ ] Remove the Reset button.
- [ ] Validate input as early as possible.
- [ ] Display errors close to the related field.
- [ ] Make error messages clear and actionable.

## Final Check

- [ ] Can the form be completed using only the keyboard?
- [ ] Is the form easy to scan?
- [ ] Can a first-time user finish it without confusion?
