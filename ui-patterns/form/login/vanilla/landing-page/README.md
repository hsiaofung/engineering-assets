## 登入與註冊表單課程重點整理

### 1. 完整的使用者流程（User Flow）

課程建立了一條完整的會員註冊流程：

```
Landing Page
      │
      ▼
Login Choice Page
 ├── Log in ──► Login Page
 └── Sign up ─► Sign-up Page
```

每個頁面都有明確的目的，讓使用者能自然完成登入或註冊。

---

### 2. 登入頁面（Login Page）設計重點

* Email 與 Password 為必填欄位
* 提供「顯示／隱藏密碼」功能（眼睛圖示）
* 提供「Forgot Password」入口
* 登入成功後進入系統

**目的：**

* 降低登入錯誤
* 提高使用便利性
* 提供完整的帳號救援流程

---

### 3. 註冊頁面（Sign-up Page）設計重點

除了基本輸入欄位之外，還加入提升 UX 的功能：

#### Tooltip（工具提示）

* 說明欄位用途
* 解釋為何需要填寫
* 減少使用者疑惑

#### Password Strength（密碼強度）

即時檢查密碼是否符合規範，例如：

* 至少 10 個字元
* 包含大寫字母
* 包含小寫字母
* 包含數字
* 包含特殊符號

讓使用者立即知道密碼是否足夠安全。

#### Password Visibility（密碼可見性）

* 可切換密碼顯示/隱藏
* 減少輸入錯誤

---

## 4. UX（使用者體驗）最佳實踐

課程將許多 UX 細節整合到表單中：

* 建立清楚的使用者流程
* 提供即時回饋（Validation）
* 降低使用者輸入錯誤
* 提供必要說明（Tooltip）
* 提升密碼安全性
* 保持一致且清晰的介面設計

---

## 5. Responsive Design（響應式設計）

採用 **Mobile First（行動裝置優先）** 的設計理念：

* 先設計手機版
* 再擴充至平板與桌面版
* 保持各裝置一致的操作體驗

---

## 6. 本課實作功能總整理

* ✅ Landing Page（著陸頁）

* ✅ Password Visibility Toggle（密碼顯示切換）
* ✅ Tooltip（工具提示）
* ✅ Password Strength Indicator（密碼強度指示器）
* ✅ Forgot Password 入口
* ✅ Responsive Layout（響應式版面）

---

## 7. 核心學習收穫

這堂課強調，**登入／註冊表單並不只是放幾個輸入框，而是一套完整的使用者體驗設計。** 一個優秀的表單應該同時兼顧：

* **流程（Flow）**：引導使用者順利完成操作。
* **易用性（Usability）**：提供清楚提示、即時回饋與方便的互動功能。
* **安全性（Security）**：透過密碼規範與強度檢查提升帳號安全。
* **一致性（Consistency）**：維持統一的版面與互動方式，降低學習成本。
* **跨裝置體驗（Responsive Design）**：確保在手機、平板與桌機上都有良好的使用體驗。

---

## 一句話總結

> **好的登入／註冊表單，不只是收集資料，而是透過完善的流程、即時回饋、安全機制與一致的介面設計，讓使用者能輕鬆、安心且順利地完成登入或註冊，進而提升整體使用體驗與轉換率。**
