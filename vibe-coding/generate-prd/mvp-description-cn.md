## 問題

你站在廚房裡，面前是一堆隨機的食材，卻不知道該煮什麼。去食譜網站搜尋，結果要滑過一堆頁面，發現自己缺了一半以上的材料。

## 解決方案

一個簡單的網頁應用：使用者輸入手上現有的食材，系統立刻根據匹配程度排序，推薦最符合的食譜，優先顯示幾乎只用到你現有材料的料理。

## 核心 MVP 流程

1. 使用者進入乾淨的首頁，看到明顯的食材輸入欄
2. 使用者一個一個加入食材（標籤式輸入(tag-style input)：輸入後按 Enter，顯示成膠曩標籤(pills)）
3. 使用者按下「尋找食譜」
4. 應用程式回傳依食材匹配百分比排序的食譜卡片列表
5. 使用者點擊卡片，查看完整食譜（材料、步驟、烹調時間、份量）

## 每份食譜需顯示的重點資訊

- 食譜名稱與圖片
- 烹調時間與難度等級
- 食材匹配情況（例如：「你有 7 種材料中的 5 種」）
- 缺少的材料會被特別標示
- 詳細頁面顯示逐步烹調說明

## MVP 範圍

- 不需要使用者帳號或登入
- 不提供收藏／儲存功能（保持無狀態）
- 食譜資料使用靜態模擬資料集（約 30–50 道）
- 單一頁面體驗，導航盡量簡化
- 支援手機並具備響應式設計

## 不在範圍內

- 使用者帳號、登入、已儲存的食譜
- 購物清單產生功能
- 飲食限制篩選（可於 MVP 後再加入）
- 外部 API 整合（先以模擬資料開始，之後再替換）
